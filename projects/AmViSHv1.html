AmViSH v1 — Embodied AI Prototype

Can a robot feel alive?

I realized that life in machines is about behavior, not complexity.

AmViSH v1 began as a Class 8 holiday assignment to build a robot model. Instead of building a static structure, the goal evolved into creating a physical embodiment of a conversational AI assistant.

This was the first time I moved from “model making” to “system thinking.”

The Core Concept

The robot’s heart acted as a life-state trigger. A magnet embedded in the heart module interacted with a digital Hall effect sensor connected to an Arduino.

State Flow

Magnet Inserted → Hall Sensor HIGH → Active State → LEDs + Beep + Voice Magnet Removed → Hall Sensor LOW → Alert State → Flash + Tone Alert

Active Mode (Heart Inserted)

  • Eye LEDs performed PWM fade-blink cycles.
  • Heart illumination revolved in a circular pattern.
  • A 10ms system beep occurred every 2 seconds.
  • Tablet-based AI voice interaction was active.

All behaviors were controlled using non-blocking timing logic based on the Arduino millis() function.

Alert Mode (Heart Removed)

  • Eye LEDs faded closed.
  • All heart LEDs flashed simultaneously.
  • The buzzer emitted an alert tone using tone().

State Architecture

The system functioned as a two-state finite state machine, transitioning between ACTIVE and ALERT states depending on the digital sensor input.

What I Learned

Physical embodiment changes how AI is perceived. Even simple timed LED behaviors can create the illusion of life when structured through clear state logic.

Complexity does not create life in machines. Behavior does.

Technical Summary

  • Arduino-based state control
  • Digital Hall sensor (HIGH/LOW trigger)
  • PWM LED fading
  • Non-blocking timing using millis()
  • Dual-speaker architecture (voice + system tone)
